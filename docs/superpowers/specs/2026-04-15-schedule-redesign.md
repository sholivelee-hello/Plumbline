# 일정(Schedule) 개편 및 체중 메뉴 추가 — 설계

## 개요

Plumbline PWA의 일정 시스템을 재구성하여:
1. 이벤트(Events)를 주간 뷰에서도 보고 등록할 수 있게 함
2. 계획 없이 실제를 직접 추가 가능하게 함
3. 블록 추가 시 날짜 선택 UI 제공
4. 월간 뷰에서 이벤트 CRUD 가능하게 함 (시간 블록은 제한)
5. **주간 템플릿** 시스템 신규 도입 — 여러 템플릿 저장/적용
6. 전체 앱을 **일요일 시작** 주간으로 마이그레이션 (현재 월요일 시작)

## 배경

현재 구조의 문제:
- 이벤트는 월간 뷰에만 표시되어 주간 플래닝 중 확인 불가
- 계획(plans)을 만들어 "완료" 해야만 실제(actuals)가 생성됨 — 직접 추가 불가
- 블록 추가 모달에 날짜 선택 UI 없음 (부모 맥락에서만 결정)
- 월간 뷰에서 이벤트를 등록/편집할 방법이 없음
- 반복되는 주간 루틴을 매번 수동으로 재입력해야 함
- 사용자는 일요일을 한 주 시작으로 사용하나 앱은 월요일 시작

## 스코프

- **V1 (이번 구현)**: 일정 개편 + 주간 템플릿 + 일요일 시작 마이그레이션
- **별도**: 체중 메뉴 추가 (분리된 후속 스펙 필요)

## 1. 페이지 헤더 구조

### 일정 페이지 상단

```
[ 일정 ]    [주간 | 월간 토글]  [📋 템플릿]  [+ 블록]  [+ 이벤트]
```

- **주간 뷰**: `+ 블록` + `+ 이벤트` 모두 표시
- **월간 뷰**: `+ 이벤트`만 표시 (블록 버튼 숨김 → 월간에서는 시간 블록 등록 불가)
- `📋 템플릿` 버튼은 두 뷰 공통 → `/schedule/templates` 이동

## 2. 주간 뷰 레이아웃

```
┌─────────────────────────────────────────────────────┐
│ [◀] 2026-04-12 ~ 2026-04-18 [▶]                      │
├─────────────────────────────────────────────────────┤
│ ── 이벤트 스트립 ──                                   │
│ ████ 여행 (수~금) ████                                │
│ 🎂 생일(월)                                           │
├──┬──────┬──────┬──────┬──────┬──────┬──────┬──────┤
│時│  일   │  월   │  화   │  수   │  목   │  금   │  토  │
│  │계획│실제│계획│실제│ ...                            │
```

### 이벤트 스트립
- 날짜 헤더 아래 가로 띠 영역
- 다일 이벤트는 해당 날짜 범위에 걸친 막대로 표시 (구글 캘린더 스타일)
- 단일 날짜 이벤트는 그 날짜 하나에만 표시
- 이벤트 클릭 → 편집 모달
- 이벤트가 없으면 스트립 영역 자체를 숨기거나 최소 높이

### 블록 그리드 (계획/실제)
- 기존 2컬럼 구조 유지 (계획 | 실제)
- **빈칸 클릭**:
  - "계획" 컬럼 빈칸 → 계획으로 블록 추가 (해당 날짜/시간)
  - "실제" 컬럼 빈칸 → 실제로 블록 추가 (해당 날짜/시간)
- **요일 순서**: 일 → 월 → 화 → 수 → 목 → 금 → 토 (일요일 시작)

## 3. 월간 뷰 동작

- 시간 블록은 표시하지 않음 (이벤트 전용)
- 이벤트만 날짜 위에 표시
- **날짜 탭 동작**:
  - 이벤트 없는 날짜 → 바로 이벤트 추가 모달 열림
  - 이벤트 있는 날짜 → 이벤트 목록 모달 (편집/삭제/추가 가능)
- 주간 뷰로 이동은 상단 "주간/월간" 토글로만

## 4. 입력 폼

### 4-1. 블록 추가 모달 (BlockForm 개편)

기존 필드 + **날짜 필드 추가** + **"실제로 기록" 체크박스 추가**

```
[날짜]     2026-04-15 (달력/날짜 피커)
[제목]     ______________
[시작]     04:00 ▾
[종료]     04:30 ▾
[색상]     ⬤ ⬤ ⬤ ⬤ ⬤ ⬤
[ ] 즐겨찾기에 저장
[ ] 실제로 기록  (기본: 계획)
[취소]  [저장]
```

- **날짜 기본값**:
  - 빈칸 클릭으로 진입: 그 칸의 날짜
  - "+ 블록" 상단 버튼으로 진입: 현재 날짜 (오늘)
- **"실제로 기록" 체크박스**:
  - 빈칸 클릭 시: 컬럼에 따라 자동 세팅 (실제 컬럼이면 자동 체크)
  - 상단 버튼 시: 수동 선택

### 4-2. 이벤트 추가/편집 모달 (신규)

```
[시작 날짜]  2026-04-15
[종료 날짜]  2026-04-17  (단일 날짜면 같은 날짜)
[제목]       ______________
[색상]       ⬤ ⬤ ⬤ ⬤ ⬤ ⬤
[메모]       ______________  (선택)
[삭제]*                    [취소] [저장]
```
- `start_time` 필드 제외 (이벤트는 날짜 기반)
- 종료 날짜 = 시작 날짜면 단일 날짜 이벤트
- `*삭제` 버튼은 편집 모드일 때만 표시

## 5. 주간 템플릿 시스템

### 5-1. 데이터 모델 (신규 테이블)

```sql
-- 템플릿 메타
CREATE TABLE weekly_templates (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- 템플릿 안의 블록들
CREATE TABLE weekly_template_blocks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  template_id UUID REFERENCES weekly_templates(id) ON DELETE CASCADE,
  day_of_week SMALLINT NOT NULL CHECK (day_of_week BETWEEN 0 AND 6),
  -- 0 = 일요일, 1 = 월요일, ..., 6 = 토요일
  start_time TIME NOT NULL,
  end_time TIME NOT NULL,
  title TEXT NOT NULL,
  color TEXT NOT NULL
);
```

- RLS: 사용자는 자신의 템플릿만 접근 가능

### 5-2. 템플릿 관리 페이지 (`/schedule/templates`)

**목록 화면**:
- 템플릿 카드 리스트
- 각 카드: 이름 + 블록 개수 + [편집] [삭제] 버튼
- 상단 "+ 새 템플릿" 버튼

**편집 화면**:
- 이름 입력 필드
- 월~일 7일 가로 그리드 (시간대 세로)
- 각 칸 클릭 → 블록 추가 모달 (제목/시간/색상)
- 기존 블록 클릭 → 편집/삭제
- 저장 버튼

### 5-3. 템플릿 생성 방식

**빈 템플릿을 처음부터 만들기** (approach B):
- 별도 편집 화면에서 7일 격자에 블록을 하나씩 추가
- 이미 사용 중인 기존 주를 기반으로 자동 복제하는 기능은 V2로

### 5-4. 템플릿 적용 흐름

```
사용자: 📋 템플릿 → 템플릿 선택 → "이 주에 적용"
                            ↓
                기존 블록 확인 (plans만, actuals는 무관)
                            ↓
          ┌───────────────┴───────────────┐
      없음                              있음
        ↓                                 ↓
   바로 적용                      확인 모달 표시
                                          │
                              ┌───────────┼───────────┐
                          덮어쓰기      합치기      취소
                              ↓             ↓           ↓
                     기존 삭제 후      기존 유지 +    아무것도
                     템플릿 적용       템플릿 추가    하지 않음
```

- 템플릿은 **plans만** 생성 (actuals는 건드리지 않음)
- 생성되는 블록은 선택한 주의 해당 요일 날짜로 매핑
- "이 주에 적용" 외 "특정 주 선택" 기능은 V1에 포함 (target week 파라미터)

## 6. 일요일 시작 마이그레이션

### 수정할 코드

- `src/lib/utils/date.ts`:
  - `getWeekStart(dateStr)`: 일요일 반환으로 변경 (현재 월요일)
  - `getWeekDates(dateStr)`: 일 ~ 토 순서로 7일 반환
- 주간 통계 컴포넌트 (이미 구현됨):
  - `src/components/basics/category-line-chart.tsx`: 요일 헤더 `["일","월","화","수","목","금","토"]`
  - `src/components/basics/check-table.tsx`: 동일
- 캘린더 히트맵: `src/components/basics/calendar-heatmap.tsx` 일요일 기준 오프셋
- 새 주간 뷰 (이번 개편): 일요일 시작

### 영향 범위
- 기존 데이터(plans, actuals, events, basics_logs)는 날짜(`date` 필드) 기반이므로 데이터 마이그레이션 불필요
- 주간 집계 함수만 주 시작 기준 바뀜 (통계는 "이번 주" 정의가 일요일 시작으로 이동)

## 8. 빈 상태 및 에러 처리

- 템플릿 0개: "아직 템플릿이 없어요. + 새 템플릿 버튼으로 만들어보세요" 안내
- 템플릿 블록 0개: 템플릿 편집 화면 격자는 비어있는 상태로 표시
- 이벤트 0개 (주간 스트립): 스트립 영역 숨김 또는 최소 높이
- 데모 데이터: 기존 앱 패턴 따라 Supabase 미구성 시 폴백

## 9. V2 / 추후 고려사항

- 템플릿을 기존 주에서 자동 생성 ("이 주를 템플릿으로 저장")
- 템플릿 공유/가져오기
- 반복 이벤트 (매주 수요일 스터디)
- 과거 주 네비게이션은 이미 가능 (주간 뷰 ◀▶)
- 체중 메뉴는 별도 스펙 (`2026-04-15-weight-menu-design.md` 예정)
