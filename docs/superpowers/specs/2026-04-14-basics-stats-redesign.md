# Basics 통계 탭 설계

## 개요

Basics(생활기본) 페이지에 "통계" 탭을 추가하여, 주간/월간 단위로 항목 달성 현황을 시각적으로 확인할 수 있는 기능을 구현한다.

## 배경

현재 basics 통계는 달성률 퍼센트와 스트릭 숫자만 보여주고 있다. 사용자는 수치형 항목의 실제 변화 추세를 그래프로 보고 싶고, 체크형 항목의 일별 달성 여부도 한눈에 파악하고 싶다. 또한 주간뿐 아니라 월간 단위로 전체적인 달성 패턴을 확인하고 싶어 한다.

## 스코프

- **V1 (이번 구현)**: 현재 주간/월간만 표시. 과거 기간 네비게이션은 V2에서 추가.
- 이유: 핵심 가치(시각적 통계 확인)를 먼저 검증하고, 네비게이션은 후속 이터레이션에서 추가한다.

## 구조

### 네비게이션

- basics 페이지 상단 탭을 **"체크" / "통계"** 로 구성 (기존 탭 이름 "체크" 유지)
- "체크" 탭: 기존 일일 체크리스트 + 현재 달성률/스트릭 카드
- "통계" 탭: 새로운 통계 화면
- 통계 탭 내에서 **주간 / 월간** 토글로 전환

## 주간(Weekly) 통계 화면

위에서 아래로 스크롤하는 단일 페이지 구성.

### 1. 전체 달성률 요약 카드

- "이번 주 달성률" 큰 퍼센트 숫자
- 아래에 영적 / 신체적 각각의 달성률 프로그레스 바
- 이번 주 월~일 기준으로 계산 (주 시작은 월요일 고정, 기존 `getWeekStart()` 사용)
- **부분 주 처리**: 오늘이 수요일이면 월~수까지만 분모로 계산 (미래 날짜 제외)

### 2. 수치형 항목 라인 차트 (영적)

- x축: 월~일 (7일, 미래 날짜는 빈칸)
- y축: 실제 수치 (분, 잔 등)
- 항목별로 라인 + 목표치 점선
- 예: "기도" 라인이 25, 30, 28, 30으로 흐르고 목표 30분 점선 표시
- 해당 카테고리에 수치형 항목이 없으면 이 섹션 숨김

### 3. 수치형 항목 라인 차트 (신체적)

- 같은 형태로 신체적 카테고리 분리 표시
- 해당 카테고리에 수치형 항목이 없으면 이 섹션 숨김

### 4. 체크형 항목 달성 현황

- 미니 테이블 형태
- 행: 각 체크형 항목
- 열: 월 | 화 | 수 | 목 | 금 | 토 | 일 | 달성률
- 각 칸에 ● (달성) / ○ (미달성) 표시
- 항목이 아직 존재하지 않았던 날짜는 "-" 표시
- 체크형 항목이 하나도 없으면 이 섹션 숨김

## 월간(Monthly) 통계 화면

마찬가지로 위에서 아래로 스크롤하는 단일 페이지 구성.

### 1. 전체 달성률 요약 카드

- "이번 달 달성률" 큰 퍼센트 숫자
- 영적 / 신체적 프로그레스 바
- 이번 달 1일~오늘 기준으로 계산 (미래 날짜 제외)
- 참고: 월 초에는 데이터가 적어 달성률이 변동폭이 클 수 있음 (정상 동작)

### 2. 캘린더 히트맵

- 달력 형태로 날짜 칸 배치 (일~토 또는 월~일 7열)
- 각 칸의 색 진하기 = 그날 전체 달성률
- 5단계 색상: 0%(빈칸/회색) → 1~25%(연한) → 26~50%(중간) → 51~75%(진한) → 76~100%(가장 진한)
- GitHub 잔디 스타일로 한눈에 패턴 파악
- 미래 날짜는 빈 칸 처리

### 3. 주차별 비교

- 1주차(1~7일) / 2주차(8~14일) / 3주차(15~21일) / 4주차(22~말일) 막대 그래프
- 주차 분리: 월의 날짜 기준 단순 분할 (ISO 주가 아닌 월 내 구간)
- 영적 / 신체적 색상을 나눠서 묶은 막대(grouped bar)
- "점점 나아지고 있는지" 주 단위 추세 파악
- 아직 시작되지 않은 주차는 표시하지 않음

### 4. 항목별 일간 테이블

- 행: 각 항목 (영적 → 신체적 순서)
- 열: 1일 ~ 말일
- 체크형: ● / ○
- 수치형: 실제 숫자 (목표 미달 시 빨간 톤, 달성 시 파란 톤)
- 항목이 존재하지 않았던 날짜: "-" 표시
- 좌우 스크롤로 30일 탐색
- 맨 오른쪽 고정 열에 해당 항목의 월간 달성률 %

## 데이터 처리 규칙

### 항목 추가

- 항목이 생성된 날짜(`created_at`)부터만 통계에 포함
- 이전 날짜는 "-" 표시 (미달성 처리 안 함)
- 달성률 계산 시 분모: 항목이 존재한 일수만

### 항목 삭제(비활성화)

- 기본적으로 통계에서 숨김
- "삭제된 항목 보기" 토글을 켜면 과거 기록 확인 가능
- 삭제된 항목은 `created_at` ~ `deactivated_at` 기간만 표시
- 흐린 스타일(opacity 등)로 현재 항목과 시각적 구분
- **전제조건**: `basics_templates` 테이블에 `deactivated_at TIMESTAMPTZ` 컬럼 추가 필요. `deactivateTemplate()` 함수에서 `is_active = false`와 함께 `deactivated_at = now()` 설정.

### 목표치 변경

- 항상 현재 목표(`target_value`) 기준으로 목표선 표시
- 과거 데이터도 현재 목표와 비교 (변경 이력 미추적)

### 수치형 항목의 "달성" 판정

- 통계에서 수치형 항목의 달성 여부는 **저장된 `value`를 현재 `target_value`와 비교**하여 실시간 판정
- `value >= target_value`이면 달성, 아니면 미달성 (`target_value`가 없으면 `value > 0`이면 달성)
- 저장된 `completed` 불리언은 사용하지 않음 (과거 목표 기준이므로 현재 목표선과 불일치 방지)

### 달성률 계산 공식

- **항목별**: `(달성한 일수 / 항목이 존재한 일수) × 100`
  - "항목이 존재한 일수" = `created_at`(또는 비활성 항목의 경우 `deactivated_at`)까지의 범위와 조회 기간의 교집합
  - 미래 날짜는 제외
- **카테고리별**: 해당 카테고리 항목들의 달성률 산술 평균 (항목 동등 가중치)
- **전체**: 모든 활성 항목의 달성률 산술 평균
- 삭제된 항목은 기본 집계에서 제외 (토글 활성화 시에도 전체 달성률에는 미포함)
- **0개 항목 시**: 달성률 표시 안 함 ("아직 항목이 없어요" 안내)

## 빈 상태 및 에러 처리

- **항목 0개**: "아직 항목이 없어요. 체크 탭에서 항목을 추가해보세요" 안내
- **특정 카테고리 항목 0개**: 해당 카테고리 섹션 숨김 (라인 차트, 프로그레스 바 등)
- **데이터 2일 미만**: 라인 차트 대신 "데이터가 쌓이면 추세를 보여드릴게요" 안내
- **로딩 중**: 스켈레톤 UI (카드/차트 영역에 펄스 애니메이션)
- **에러/오프라인**: 기존 앱 패턴 따라 데모 데이터로 폴백

## 기술 참고

### 현재 데이터 모델

- `BasicsTemplate`: id, category("spiritual"|"physical"), title, type("check"|"number"), unit, target_value, is_active, created_at
- `BasicsLog`: id, template_id, date, completed, value, completed_at

### 필요한 스키마 변경

- `basics_templates` 테이블에 `deactivated_at TIMESTAMPTZ DEFAULT NULL` 컬럼 추가
- `deactivateTemplate()` 함수 수정: `deactivated_at = new Date().toISOString()` 함께 설정

### 차트 라이브러리

- **recharts** 사용 (이미 `package.json`에 설치됨: `recharts ^3.8.1`)
- 캘린더 히트맵: CSS Grid + 동적 배경색으로 직접 구현 (recharts 불필요)

### 신규 훅

기존 `useBasicsStats` 훅은 다음이 부족하므로 **신규 훅을 생성**한다 (기존 훅은 수정하지 않음):

- 기존 훅의 한계:
  - `day_start_time` / `getLogicalDate()` 미사용
  - `value` 필드 미조회 (수치형 차트 불가)
  - `type`/`unit`/`target_value` 미조회
  - 비활성 템플릿 조회 불가

- 신규 훅:
  - `useWeeklyStats(dayStartTime)`: 이번 주 7일간 데이터. 템플릿 + 로그 + value 포함
  - `useMonthlyStats(dayStartTime)`: 이번 달 데이터. 템플릿 + 로그 + value 포함
  - 두 훅 모두 비활성 템플릿을 선택적으로 포함하는 `includeInactive` 파라미터 지원

### 주간 범위 결정

- 주 시작은 월요일 고정 (기존 `getWeekStart()` 유틸 사용)
- 논리적 날짜는 `day_start_time` 기준 (`getLogicalDate()` 사용)

### 반응형

- 모바일: 차트 가로 스크롤, 월간 테이블 좌우 스와이프
- 데스크톱(lg+): 사이드바 레이아웃과 조화, 차트 넉넉한 너비
